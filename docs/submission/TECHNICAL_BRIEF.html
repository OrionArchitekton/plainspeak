01 The problem

Ordinary people sign leases, accept medical plans, and click I agree on terms they do not actually understand. The language is written to be understood by the party that wrote it, not the person it affects. That asymmetry quietly costs people money, rights, and peace of mind. Plainspeak closes the comprehension gap, and does it in a way that respects the reader: it tells you what affects you, not just a generic summary.

02 How it works

One Next.js app, one server route, one structured model call. The Anthropic API key never reaches the browser; the model is constrained by a system prompt and a JSON contract, and the reply is parsed defensively so a malformed response degrades gracefully instead of crashing.

Browser Next.js 15 / React 19 │ POST /api/explain { document, readingLevel, situation } ▼ Server route (nodejs runtime · key stays server-side · optional per-IP rate limit) │ system prompt = persona + reading level + "not legal/medical advice" guardrail + JSON contract ▼ Anthropic Claude claude-sonnet-4-6 · temperature 0 (deterministic, repeatable) │ reply → defensive JSON parse (recovers code-fenced / messy) → sort clauses high→low ▼ Three cards In plain words · What affects you (ranked) · Questions to ask

03 The output that matters

Not a summary, three answers. The middle one is the differentiator: the specific clauses that affect the reader, ranked so the traps surface first.

HIGH
Your $4,800 deposit is non-refundable if you leave before 12 months, even with no damage.
MEDIUM
Rent can jump 12% at renewal with only 30 days' notice, less than the 60 you need to leave.
LOW
The landlord may enter with 24 hours' notice; the document never defines valid notice.

Then the exact questions to ask the landlord, clinician, or company, so the reader walks in informed instead of intimidated. Reading level is a control: Like I'm 10, Plain English, or Full detail.

04 Safe and deterministic by construction

  • Guardrail in the model's own instructions — "never invent facts; this is not professional advice." The UI repeats it.
  • Temperature 0 — the same document yields the same explanation, every time.
  • No persistence — documents are processed and returned, never stored. The only write is an optional rate-limit counter (IP + count), never content.

05 Engineering rigor

The prompt-build and parse logic is a pure module, unit-tested without a network. The build is verified live, not just compiled, and every load-bearing claim survived an independent adversarial review before submission.

Shaped by adversarial review

"You say clauses are 'ranked by severity', but nothing sorts them."
Correct. Added a stable high→medium→low sort in the parser, so the claim is true on the live path, not just in the demo fixture.
"The rate limit shipped but never returns 429."
Vercel's Upstash integration injects KV_REST_API_*, not UPSTASH_REDIS_REST_*. The limiter now accepts both, verified returning 429 under a concurrent burst.
